# semiconduct-kor / kr-rally-relay

`semiconduct-kor` is a KRW-only live sleeve whose operator-facing strategy name
is **kr-rally-relay**. The id is intentionally preserved so existing account
routes, reports, control commands, and runtime state keep their lineage.

Runtime configs:

```text
configs/runtime/live_multi_sleeve.json
configs/runtime/semiconduct_kor_sleeve.json
configs/runtime/semiconduct_kor_shadow.json
```

Executable flow:

```text
KRX rally universe
  -> daily agent target selection
  -> no in-process alpha
  -> complete target portfolio construction
  -> basic long-only risk
  -> standard limit execution intents
```

Workspace layout:

```text
sleeves/semiconduct-kor/
  alphas/          # legacy/research modules, not live-wired by default
  selections/
    agent_narrative_target.py
    operational_symbols.py
  portfolios/
    agent_narrative_target.py
    samsung_buy_only.py       # rollback/reference
  risks/
    basic.py
  executions/
    immediate.py
```

The live profile reads:

```text
data/operator-targets/semiconduct-kor/latest_target.json
```

The target artifact is generated by:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools/build_semiconduct_kor_live_target.ps1
```

The target builder uses stored KRX news evidence and local daily bars with an
08:00 Asia/Seoul decision cutoff. It writes both an auditable dated target under
`sleeves/semiconduct-kor/agent_state/pseudo_portfolios/narrative_relay_v1/` and
the live `latest_target.json` consumed by runtime.

Useful checks:

```powershell
py -3 -m leaps_quant_engine.cli runtime-config-validate configs/runtime/live_multi_sleeve.json
py -3 -m leaps_quant_engine.cli runtime-preflight configs/runtime/live_multi_sleeve.json --sleeve-id semiconduct-kor --summary-only
py -3 -m leaps_quant_engine.cli runtime-backtest-minute configs/runtime/semiconduct_kor_agent_narrative.json --sleeve-id semiconduct-kor --minute-feed data/replay/leaps_krx_20260421_20260515_merged_stock_etf_minute.csv --start 2026-04-21 --end 2026-05-16 --daily-source parquet-daily --summary-only
```
